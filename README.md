# Chatlog Viewer

一个用于查看 Codex `jsonl` 历史记录的本地桌面工具。

它的核心目标很简单：

- 按工作路径整理聊天记录
- 在同一路径下按时间点查看对话
- 用户消息默认展开
- AI 回复默认折叠
- 支持复制当前时间点的完整对话
- 支持手动选择 `sessions` 数据目录

## 功能

- 自动扫描 `jsonl` 文件
- 自动提取 `cwd`，建立 `路径 -> 时间点(jsonl)` 映射
- 一个 `jsonl` 如果命中多个工作路径，会同时归到多个路径下
- 右侧支持复制当前时间点的完整内容
- 支持修改数据目录
- 左侧路径支持置顶、自定义多选、资源管理器打开、导出历史记录、CMD 打开
- 支持在单个 Windows Terminal 标签内慢速打开多个窗格
- 默认建议目录为：
  - `C:\Users\当前用户\.codex\sessions`

## 使用方式

这个项目支持两种使用方式。

### 方式一：直接下载可执行文件

如果你只是想直接使用，不想自己配置开发环境，可以直接下载已经打包好的便携版：

- `dist/Chatlog Viewer 0.1.0.exe`

下载后直接双击打开即可使用。

首次打开后：

1. 确认左侧显示的数据目录
2. 如果默认目录不对，点击“选择目录”
3. 选中真正的 `sessions` 目录后即可使用

### 方式二：下载源码后自己运行或打包

如果你想自己改代码、自己运行、或者自己重新打包，可以下载源码。

## 适用场景

适合这类使用方式：

- 同时开多个 CMD / PowerShell 窗口
- Codex 会生成很多 `rollout-*.jsonl`
- 想按项目路径回看历史对话
- 不希望前端一次加载所有内容导致卡顿

## 使用说明

1. 打开应用
2. 确认左侧“数据目录”
3. 如果默认目录不对，点击“选择目录”
4. 在左栏选择路径
5. 在中栏选择某个时间点
6. 在右栏查看对话，或点击“复制当前对话”

## 数据目录说明

应用默认会优先使用：

- 用户自己配置过的目录

如果没有配置，则使用默认建议目录：

- `C:\Users\当前用户\.codex\sessions`

## 当前实现说明

当前版本采用：

- 路径维度聚合
- 时间点维度展示
- 时间点第一版直接按 `jsonl` 文件分组

也就是说：

- 先选路径
- 再看这个路径下命中的 `jsonl`
- 每个 `jsonl` 作为一个时间点入口

这是为了尽量减少复杂度，并保证加载速度。

## 已知限制

- 目前“时间点”按整份 `jsonl` 展示，不做更细粒度切片
- 某些 `jsonl` 如果内部包含多个 `cwd`，会同时出现在多个路径下
- 当前复制内容会包含用户消息、AI 回复、命令和命令输出
- 目前没有全文搜索全部时间点的功能
